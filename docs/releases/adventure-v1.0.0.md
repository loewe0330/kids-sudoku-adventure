# 数独探险家闯关系统 v1.0.0

发布日期：2026-07-16

## 本阶段内容

- 儿童档案创建仅需要昵称、年级和可选头像；正式能力等级由有效练习逐步评估。
- 年级仅用于首次练习冷启动推荐，能力等级与闯关进度相互独立。
- 自由练习提供每日推荐、巩固和挑战入口，并保留现有自主选题方式。
- 普通闯关每关最多提交两次。第一次错误后可以继续修改，第二次提交决定通关或最终失败。
- 最终失败流程支持重新挑战、返回当前大关和查看解题方法，不改变历史星级或解锁下一关。
- 第一次提交错误后可以选择一次分层解题引导，引导不会直接给出完整答案。
- 每个孩子每天有 3 次免费引导；免费次数用完后，每天最多使用 3 颗可用星星兑换引导。
- 历史累计星星由各关历史最高星级汇总，兑换仅增加累计消费星星，两类数据相互隔离。
- 引导消费使用唯一 `operationId` 去重，避免双击、重试或跨设备同步导致重复扣除。
- 闯关秘籍通过 3 道验证题确定快速通关起点，不直接修改正式能力等级或历史关卡成绩。
- 同一账号、同一 `childId` 的孩子资料、练习记录、闯关进度、引导次数及星星消费支持跨设备同步。
- 离线数据恢复联网后合并补传；完成记录、历史最高星级、已解锁状态和消费操作不会被旧设备数据回退。

- 网页固定尺寸：1440 x 900，通过。
- iPad 固定尺寸：880 x 1035，自动化与真机验收通过。
- 手机固定尺寸：361 x 612，自动化与真机验收通过。
- 同账号、同孩子的 iPad 与手机互传通过。
- 自动测试：37 个测试文件、212 项测试通过。
- TypeScript 检查和 Vite 生产构建通过。
- `git diff --check` 通过。

## 已知限制

- 云同步依赖网络与 Netlify 服务；网络不可用时保留本机数据，恢复联网后再同步。
- 浏览器缓存被主动清除前，应确保最近一次云同步成功。
- GitHub Pages 暂时保留为备用入口，正式服务与云函数以 Netlify 版本为准。
