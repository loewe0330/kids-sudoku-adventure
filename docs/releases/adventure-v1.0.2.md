# 数独探险家 v1.0.2

发布日期：2026-07-17

## 本次修改

- 统一重构孩子首页、自由练习、成长报告、设置和孩子选择页的森林探险视觉。
- 新增共享应用头部、五项底部导航、功能入口卡、数据摘要卡、折叠详情、设置行和开关组件。
- 首页优先展示今日任务、当前闯关地图和三个核心入口；小游戏收纳到次级折叠区。
- 自由练习保留每日推荐、题库、打印和自选配置，将低频选项收纳到次级区域。
- 成长报告将详细数据分成四个默认折叠的洞察模块，首屏只保留称号、星星、完成率和进度。
- 设置页集中展示同步状态、孩子设置、本地界面偏好与家长密码入口。
- 本补丁不修改闯关解锁、星星计算、引导消费、云同步模型、儿童档案结构或 Netlify 云函数。

## 图片与性能

- 新增 41 张本地 WebP 界面素材，生产资源总计 519,088 bytes（约 507 KiB）。
- 最大单张为成长页主视觉，48,376 bytes（约 47 KiB）；没有 2 MiB 以上生产图片。
- 所有页面图片通过统一素材映射和 `AssetImage` 组件加载，包含懒加载、稳定容器和失败回退。
- 首页首屏图片使用 eager 加载，次级入口与折叠详情使用 lazy 加载。
- 上一版本的闯关 mobile/tablet/desktop 响应式图片策略保持不变，本补丁没有重新编码或替换关卡原图。

## 三端验收

- 网页 1440 x 900：首页、关卡详情、普通答题和设置页无横向溢出、破图或阻断交互。
- iPad 880 x 1035：首页、关卡详情、普通答题和设置页布局完整，可见按钮触控高度不低于 44px。
- 手机 361 x 612：核心入口优先显示，固定导航不遮挡关键操作，无横向溢出或破图。
- 普通闯关、两次提交、失败流程、解题引导和奖励弹窗继续由既有回归测试覆盖。

## 自动化检查

- `npm test`：38 个测试文件、218 项测试全部通过。
- TypeScript 与 Vite 生产构建：通过。
- `git diff --check`：通过。

## 已知限制

- 护眼、震动和学习提醒属于当前设备的界面偏好，不写入儿童核心档案，也不参与跨设备同步。
- 云同步和引导消费依赖 Netlify 服务；离线时学习记录先保留在当前设备，恢复网络后再同步。
- 登录页在已有有效会话时会按现有规则跳转到孩子选择页。
- GitHub Pages 继续保留为备用入口，正式服务与云函数以 Netlify 版本为准。
