# 数独探险家闯关系统 v1.0.1

发布日期：2026-07-17

## 本次修改

- 统一优化孩子首页、自由练习、成长页和 11 个关卡详情页的字体、信息层级与响应式布局。
- 精简首屏说明，将主要入口和关键操作置于更容易发现的位置。
- 修复矮屏手机上的内容重叠、过度留白和关键按钮位置问题。
- 本补丁不修改闯关规则、星星规则、引导规则、云同步模型、数据结构或 Netlify 云函数。

## 图片优化

- 11 张关卡插画按设备输出为 WebP：手机宽约 720px、iPad 宽约 1400px、桌面保持接近原始宽度。
- 手机、iPad、桌面默认质量分别为 86、88、90，并保留代表图三档质量样本供人工对比。
- 页面使用 `picture` 和媒体条件只加载当前设备对应资源；关卡详情按路由只渲染当前关卡图片，不会同时请求 11 张插画。
- 图片容器使用稳定的宽高比，减少加载时布局跳动；原始 PNG 保留在源码资源中，并在生产构建结束时从 `dist` 排除。

## 性能结果

- 原始 11 张 PNG 加闯关地图约 26.9 MiB。
- mobile 全套约 0.84 MiB。
- tablet 全套约 2.28 MiB。
- desktop 全套约 4.23 MiB。
- 优化资源最大单张约 468 KiB，生产图片中没有 2 MiB 以上文件。

## 验收结果

- 网页 1440 x 900：主要入口、关卡地图和详情页布局通过，无横向溢出。
- iPad 880 x 1035：加载 tablet 图片，触控区域和页面滚动通过。
- 手机 361 x 612：加载 mobile 图片，关键按钮可见，无横向溢出。
- 11 个关卡共用同一详情模板并通过组件测试；人工视觉抽查覆盖 L1、L6、L11。
- 自动测试：37 个测试文件、215 项测试通过。
- TypeScript 检查与 Vite 生产构建通过。
- `git diff --check` 通过。

## 已知限制

- 图片清晰度以正常页面显示为验收基准，不以 400% 放大对比作为上线阻断。
- 云同步与引导消费依赖 Netlify 服务；离线时先保留本地状态，恢复网络后再同步。
- GitHub Pages 继续保留为备用入口，正式服务与云函数以 Netlify 版本为准。
