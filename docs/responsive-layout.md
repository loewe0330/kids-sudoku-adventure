# 响应式布局说明

## 断点

设备识别由 `src/platform/web/webDeviceAdapter.ts` 统一定义：

- `phone`：宽度 `< 768`
- `tablet-portrait`：`768 <= 宽度 < 1100` 且高度大于等于宽度
- `tablet-landscape`：`900 <= 宽度 < 1366` 且宽度大于高度
- `desktop`：宽度 `>= 1100`，或更大桌面窗口

CSS 主要在 `src/styles/responsive.css` 中实现。

## 手机

目标尺寸：360×640、375×667、390×844、414×896。

- 页面宽度使用 `calc(100vw - 20px)`，避免横向溢出。
- 做题页单列布局。
- 棋盘使用 `--board-phone`，不超过屏幕宽度。
- 数字键盘固定在棋盘下方附近，按钮不小于 44px。
- 工具按钮两列排布，方便触控。
- 顶部信息压缩，方法和提示面板保留折叠能力。

## iPad 竖屏

目标尺寸：768×1024、820×1180、1024×1366。

- 做题页保持单列，棋盘居中并使用更大的 `--board-tablet`。
- 数字键盘按钮加大。
- 表格保持横向滚动容器。
- 首页和地图卡片保持清晰扫描。

## iPad 横屏

目标尺寸：1024×768、1180×820、1366×1024。

- 做题页优先左右布局：左侧棋盘，右侧信息、数字键盘和操作。
- 闯关地图支持左右感布局：大关卡片更紧凑，小关节点保留可点击尺寸。
- 管理者后台表格保留桌面读表能力。

## 桌面

目标尺寸：1280×720、1440×900、1920×1080。

- 页面最大宽度提升到 1320px。
- 做题页使用三栏倾向：信息、棋盘、数字键盘/工具。
- 棋盘使用 `--board-desktop`，避免桌面端浪费空间。
- 管理者后台继续使用表格。

## 打印

打印规则放在 `src/styles/print.css`：

- 打印时隐藏导航和按钮。
- 保持黑白清晰边框。
- 保留一页 1/2/4 题和答案页结构。
